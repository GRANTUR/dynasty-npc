# DynastyNPC

AI-powered NPC plugin for Paper Minecraft 1.21.11. Creates immersive, conversational villager NPCs using Ollama LLMs with persistent memory, set in a Song Dynasty China theme.

## Overview

DynastyNPC spawns villager entities that players can talk to using natural language. Each NPC has a distinct personality, profession, and area of expertise. Dialogue is generated by an LLM (via Ollama), and the plugin maintains a three-tier memory system so NPCs remember past conversations across server restarts.

The plugin is designed for production Kubernetes environments. All LLM calls are fully async and never block the main server thread. A built-in Prometheus metrics endpoint provides observability into LLM latency, token usage, and error rates.

## Features

- LLM-powered dialogue via Ollama with configurable model and temperature
- Three-tier memory system: short-term context, disk-persisted JSON logs, and LLM-generated summaries
- Prometheus metrics endpoint (requests, latency histograms, tokens, errors, active requests)
- Fully async HTTP calls to Ollama using CompletableFuture
- Entity protection: NPCs are invulnerable and suppress trade menus
- Admin and player commands with tab completion
- Hot-reloadable configuration
- Default NPCs created on first run

## NPCs

Three NPCs are included by default, each with a unique personality and villager profession:

| NPC | Color | Profession | Expertise |
|-----|-------|------------|-----------|
| Scholar Wu | Gold | Librarian | Enchanting, brewing, book-crafting, rare materials (lapis, amethyst, echo shards) |
| Merchant Chen | Green | Cartographer | Trading, farming, food, ores, ingots, villager economics |
| General Zhao | Red | Weaponsmith | Combat, armor, weapons, mob weaknesses, fortress defense |

All NPCs stay in character as residents of the High Palace -- Center Kingdom, a Minecraft server set in Song Dynasty China.

## Prerequisites

- Paper Minecraft 1.21.11 or compatible fork
- Java 21
- Maven 3.6+
- Ollama instance accessible from the server (default model: `llama2:13b-chat-q4_K_M`)

## Build and Deploy

### Build

```bash
cd /home/opc/dynasty-npc
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-21.0.10.0.7-1.0.1.el9.aarch64
mvn clean package
```

This produces `target/dynasty-npc-1.0.0.jar`.

### Deploy to Kubernetes

```bash
POD=$(kubectl get pods -n minecraft -l app=minecraft -o jsonpath='{.items[0].metadata.name}')
kubectl cp target/dynasty-npc-1.0.0.jar minecraft/$POD:/data/plugins/DynastyNPC.jar -c minecraft
kubectl rollout restart deployment/minecraft -n minecraft
```

### CI/CD

Tag pushes (`v*`) trigger the shared `plugin-build.yml` reusable workflow from the `fortress-minecraft` repository, which builds and deploys the plugin via GitHub Actions and ArgoCD.

## Configuration

The plugin generates a `config.yml` on first run. Key settings:

```yaml
ollama:
  host: "http://ollama-relay.minecraft.svc.cluster.local:11434"
  model: "llama2:13b-chat-q4_K_M"
  max-tokens: 100
  timeout: 30

npc:
  talk-radius: 10      # blocks
  cooldown: 3           # seconds between messages per player

memory:
  enabled: true
  max-recent-messages: 6         # short-term context window
  summarize-after: 20            # messages before triggering LLM summarization
  summarize-interval: 1800       # seconds between summarization sweeps
  max-summary-length: 500

metrics:
  enabled: true
  port: 9225
```

NPC personalities are defined in the `default-npcs` section and can be customized. New NPCs created via `/dnpc create` are persisted to `plugins/DynastyNPC/npcs.yml`.

## Metrics

When enabled, the plugin runs an embedded HTTP server exposing Prometheus-format metrics on the configured port (default 9225).

**Exposed metrics:**

| Metric | Type | Description |
|--------|------|-------------|
| `dynastynpc_requests_total` | Counter | Total LLM requests, labeled by NPC |
| `dynastynpc_request_duration_seconds` | Histogram | LLM response latency distribution |
| `dynastynpc_tokens_total` | Counter | Total tokens generated |
| `dynastynpc_errors_total` | Counter | LLM errors, labeled by NPC |
| `dynastynpc_active_requests` | Gauge | Currently in-flight LLM requests |

These metrics are scraped by Prometheus and visualized in the "AI Operations - DynastyNPC" Grafana dashboard, which tracks request rates, P50/P95/P99 latency, tokens per minute, and error rates broken down by NPC.

## Architecture

```
Player ──/ask──> ChatListener / CommandHandler
                        │
                        ├── MemoryManager (3-tier: in-memory, disk JSON, LLM summaries)
                        │
                        └── OllamaClient (async HTTP, CompletableFuture)
                                │
                                └── K8s Service ──> socat relay ──> Tailscale ──> Ollama (PC)
```

**Source files** (`src/main/java/com/highpalace/dynastynpc/`):

| File | Responsibility |
|------|---------------|
| `DynastyNPC.java` | Main plugin class, wires all components |
| `OllamaClient.java` | Async HTTP client for Ollama with metrics instrumentation |
| `NPCManager.java` | NPC data, villager spawning, YAML persistence |
| `MemoryManager.java` | Three-tier persistent memory (short-term, disk, LLM summary) |
| `MetricsCollector.java` | Thread-safe Prometheus metric counters and histograms |
| `MetricsServer.java` | Embedded JDK HttpServer serving /metrics |
| `ChatListener.java` | AsyncChatEvent handler with memory and metrics integration |
| `CommandHandler.java` | /dynastynpc and /ask command routing with tab completion |
| `NPCInteractListener.java` | Prevents villager trade menus and damage on NPC entities |
